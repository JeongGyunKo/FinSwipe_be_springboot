# 에이전트형 뉴스 큐레이션 — FE 핸드오프

카드 피드가 **유저 행동 신호로 개인화 재정렬**되고, 각 카드에 **추천 이유**를 붙일 수 있게 됐어요. 엔드포인트 2개만 추가로 붙이면 됩니다. (피드 조회 `GET /news/latest`는 그대로)

## 동작 개요

- 피드(`GET /news/latest`)는 이제 "오늘 파워 상위 후보"를 **유저별로 재정렬**해 30개 반환.
  - 점수 = 뉴스 파워 + 종목 친밀도(좋아요/싫어요/읽음/체류로 학습).
  - **탐색 쿼터 6개**는 개인화와 무관한 순수 파워 뉴스 → 취향에 갇히지 않게(필터버블 방지).
  - 신호가 없는 신규 유저는 자동으로 순수 파워순(기존과 동일).
- 응답 형식·호출 방법은 **변화 없음**. FE는 아래 2가지만 추가하면 개인화가 더 똑똑해져요.

## 1) 행동 신호 보내기 (개인화 학습 재료)

이미 잡히는 신호: **좋아요**(`POST /news/{id}/like`), **싫어요**(`.../dislike`), **읽음**(`.../read`) — 기존 그대로.

추가로 **노출·체류시간**을 보내주면 개인화가 정교해져요:

```
POST /events/batch
Authorization: Bearer <JWT>
Content-Type: application/json

{
  "events": [
    { "type": "dwell", "article_id": "uuid", "dwell_ms": 4200 },
    { "type": "impression", "article_id": "uuid" },
    { "type": "open", "article_id": "uuid" }
  ]
}
```
- `type`: `impression`(카드 노출) | `dwell`(카드 체류, `dwell_ms` 필수) | `open`(상세 열람) | `skip`(빠르게 넘김)
- **`dwell_ms`가 핵심 신호**예요. 카드가 화면에 떠있던 시간(ms)을 재서 보내주세요. (오래 본 종목 = 관심 ↑)
- 모아서 배치로(예: 화면 이탈/주기적 flush 시), 한 번에 최대 30건.
- 응답: `{ "ok": true, "saved": 3 }`

> 이게 없어도 좋아요/싫어요/읽음만으로 개인화는 돌아가요. 다만 **체류시간을 보내야 "오래 본 종목" 신호가 반영**됩니다.

## 2) 추천 이유 붙이기 (선택)

피드를 그린 뒤, 화면에 보이는 카드 id를 모아 이유를 지연 호출로 받아 붙일 수 있어요:

```
POST /analysis/reco-reasons
Authorization: Bearer <JWT>
Content-Type: application/json

{ "article_ids": ["uuid1", "uuid2", "..."] }
```
응답:
```json
{ "reasons": { "uuid1": "관심 종목 흐름과 맞닿은 강세 신호", "uuid2": "리스크 관점에서 짚어볼 악재" } }
```
- 최대 30개. 유저 성향에 맞춰 생성되고 **(기사,성향)별 캐시**라 보통 즉시 응답(첫 생성만 잠깐).
- id가 응답에 없으면 이유 미생성 → 그냥 이유 없이 렌더하면 됩니다.
- **피드 로딩을 막지 마세요.** 카드부터 그리고, 이유는 도착하는 대로 채워 넣는 방식 권장.

## 정리 — FE 할 일
1. (권장) 카드 체류시간 측정 → `POST /events/batch`로 `dwell` 이벤트 전송. 개인화 품질의 핵심.
2. (선택) 피드 렌더 후 `POST /analysis/reco-reasons`로 추천 이유 받아 카드에 표시.
3. `GET /news/latest` 호출 코드는 그대로. 개인화·탐색·콜드스타트는 서버가 알아서 처리.
